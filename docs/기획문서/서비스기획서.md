# 가계부 기반 저축 운영 추천 서비스 기획서

## 1. 문서 개요

본 문서는 가계부 기반 저축 운영 추천 서비스의 초기 기획 방향을 정의하기 위한 문서이다.  
본 서비스는 사용자의 수입, 지출, 비상금 보유 현황, 저축 목표, 저축 성향 등의 정보를 바탕으로 현금흐름을 분석하고, 이에 적합한 저축 운영 방식을 제안하는 것을 목적으로 한다.

본 문서의 범위는 다음과 같다.

- 서비스의 목적 및 방향 정의
- 초기 서비스 범위 확정
- 입력 정보 및 활용 목적 정의
- 핵심 계산 및 추천 구조 정의
- 예외 상황 및 대응 원칙 정의
- 화면 단위 기획의 기준점 정리

## 2. 서비스 정의

### 2.1 한 줄 정의

가계부 데이터를 기반으로 사용자의 현금흐름을 분석하고, 상황에 맞는 저축 운영 방식을 추천하는 개인 재무 코치형 서비스

### 2.2 서비스 목적

기존 가계부 서비스는 수입과 지출 기록, 카테고리별 통계, 월간 소비 리포트 제공에는 강점을 가지지만, 사용자가 실제로 필요로 하는 행동 지침까지 연결하지 못하는 경우가 많다.

본 서비스는 단순 기록을 넘어 다음과 같은 질문에 답하는 것을 목표로 한다.

- 사용자는 이번 달에 실제로 얼마를 저축할 수 있는가
- 남는 자금을 어떤 방식으로 배분하는 것이 적절한가
- 현재 상황에서 비상금 확보가 우선인지, 예적금 중심 운영이 적절한지, ISA 또는 적립식 운용을 고려할 단계인지
- 사용자의 현금흐름과 성향을 반영할 때 무리 없는 저축 구조는 무엇인지

즉, 본 서비스는 가계부 기록을 실질적인 자산관리 행동으로 연결하는 것을 핵심 가치로 한다.

## 3. 서비스 범위

### 3.1 초기 버전에 포함하는 범위

- 수입 및 지출 기반 현금흐름 분석
- 월 저축 가능액 계산
- 저축 성향 기반 저축 방식 추천
- 비상금, 파킹통장, 예적금, ISA, 적립식 운용 등 자금 배분 가이드 제공
- 이번 달 실행 가능한 저축 플랜 제안

### 3.2 초기 버전에서 제외하는 범위

- 개별 종목 추천
- 실시간 금융상품 비교
- 자동 투자 실행
- 수익률 예측
- 자산 전체 포트폴리오 관리
- 세금, 보험, 대출까지 포함한 종합 재무설계

### 3.3 서비스 운영 원칙

- 본 서비스는 종목 추천 서비스가 아니라 저축 운영 방식 추천 서비스로 정의한다.
- 본 서비스는 사용자의 입력 정보를 바탕으로 일반적인 재무 가이드를 제공한다.
- 추천 결과는 상품 판매나 투자 권유가 아닌 자금 운영 방향 제안에 초점을 둔다.

## 4. 핵심 사용자 문제

본 서비스가 해결하고자 하는 핵심 문제는 다음과 같다.

- 사용자는 자신의 월별 잉여자금을 명확히 파악하지 못한다.
- 사용자는 남는 자금을 어떤 방식으로 배분해야 하는지 판단하기 어렵다.
- 사용자는 가계부를 작성하더라도 기록 결과를 저축과 자산관리 행동으로 연결하지 못한다.
- 사용자는 투자보다 먼저 어떤 재무 우선순위를 가져야 하는지 알기 어렵다.

따라서 본 서비스는 기록 중심 경험이 아닌 실행 중심 경험을 제공해야 한다.

## 5. 핵심 가치 제안

본 서비스의 핵심 가치 제안은 다음과 같다.

- 기록에 머무르지 않고 추천까지 이어지는 가계부 경험 제공
- 사용자의 현금흐름과 목표를 반영한 현실적인 저축 운영 가이드 제공
- 종목이 아닌 방식 중심의 추천으로 이해하기 쉬운 자산관리 경험 제공
- 복잡한 금융지식 없이도 실행 가능한 월간 저축 플랜 제공

예를 들어, 서비스는 단순히 소비가 많다는 사실을 보여주는 데 그치지 않고, 사용자의 잉여자금과 비상금 상태를 바탕으로 구체적인 자금 배분 구조를 제안해야 한다.

- 일부 값은 대략적인 추정 입력도 허용한다.
- 추천 정확도보다 입력 완료율을 우선 고려한다.

## 7. 핵심 계산 구조

### 7.1 기본 계산 항목

- 월 잉여자금 = 월 실수령액 - 월 고정비 - 월 변동지출
- 권장 저축 가능액 = 월 잉여자금 중 생활 안정성을 해치지 않는 범위의 금액
- 비상금 부족 여부 = 최소 생활비 기준 n개월치 대비 부족분
- 추천 방식 분기 = 비상금 상태 + 저축 성향 + 목표 기간 + 현금흐름 안정성

### 7.2 결과값의 기본 구성

추천 결과는 단일 숫자가 아니라 다음과 같은 구조로 제공한다.

- 안전 저축액
- 권장 저축액
- 도전 저축액

해당 구조는 사용자가 본인의 상황에 맞게 보수적 선택과 적극적 선택 사이에서 판단할 수 있도록 하기 위한 것이다.

### 7.3 입력값과 계산의 연결

- 월 실수령액, 월 고정비, 월 변동지출: 월 잉여자금 계산
- 현재 비상금 보유액: 비상금 우선 확보 여부 판단
- 저축 목표, 목표 기간: 단기/중기/장기 자금 분리
- 저축 성향: 예적금, 유동성 자산, 적립식 운용 비중 조절
- 월급일: 저축 실행 타이밍 추천

## 8. 추천 구조

### 8.1 추천의 기본 원칙

- 추천 단위는 종목이 아니라 자금 운영 방식으로 정의한다.
- 추천은 사용자의 성향뿐 아니라 현금흐름 안정성 및 비상금 상태를 함께 반영한다.
- 현금흐름이 불안정하거나 비상금이 부족한 경우, 투자형 방식보다 유동성 확보를 우선한다.

### 8.2 추천 결과 구성 요소

결과 화면은 최소한 아래 항목을 포함해야 한다.

- 이번 달 현금흐름 요약
- 월 저축 가능액
- 추천 저축 방식
- 자금 배분 비율 또는 금액
- 추천 이유
- 주의 포인트
- 이번 달 실행 액션

### 8.3 추천 결과 예시 형식

- 월 저축 가능액: 45만 원
- 추천 구조: 파킹통장 15만 원 / 적금 20만 원 / ISA 적립 10만 원
- 추천 이유: 비상금이 충분하지 않지만 현금흐름은 안정적이므로 유동성과 저축을 병행하는 구조가 적절함
- 실행 액션: 월급일 다음날 자동이체 설정

## 9. 예외 상황 및 대응 원칙

### 9.1 주요 예외 상황

- 월 잉여자금이 0원 이하인 경우
- 고정비 비중이 과도하게 높은 경우
- 비상금이 전혀 없는 경우
- 목표 수준이 현재 현금흐름 대비 과도한 경우
- 저축 성향과 실제 현금흐름이 맞지 않는 경우
- 수입이 불규칙한 경우

### 9.2 예외 상황 대응 원칙

- 일반 추천 대신 별도의 경고 또는 보수적 안내를 제공한다.
- 투자형 운용보다 지출 구조 조정 또는 유동성 확보를 우선 제시한다.
- 추천 불가가 아니라 우선순위 전환 형태로 안내한다.

### 9.3 예외 메시지 예시

- 현재는 저축 확대보다 지출 구조 조정이 우선입니다.
- 비상금이 부족한 상태이므로 투자형 운영보다 유동성 확보가 우선입니다.
- 현재 목표 금액은 현금흐름 대비 과도하므로 단계적 목표 재설정이 필요합니다.

## 10. 화면 단위 기획 기준

### 10.1 필수 화면

- 온보딩
- 기본 정보 입력
- 지출 정보 입력
- 성향 및 목표 입력
- 결과 리포트
- 월별 현황 대시보드

### 10.2 화면별 정의 항목

각 화면은 아래 항목을 기준으로 상세 기획한다.

- 화면 목적
- 입력 항목
- 노출 정보
- 다음 액션
- 예외 처리

### 10.3 결과 화면의 역할

결과 화면은 단순 조회 화면이 아니라, 사용자가 이번 달 저축 계획을 이해하고 바로 실행할 수 있도록 만드는 실행 유도 화면이어야 한다.

핵심 정보는 다음과 같다.

- 월 잉여자금
- 권장 저축액
- 추천 운영 방식
- 실행 액션

예상 CTA는 다음과 같다.

- 추천 계획 저장
- 다음 달 비교 보기
- 자동 실행 일정 설정

## 11. 신뢰 및 정책 원칙

본 서비스는 금융 관련 판단에 영향을 줄 수 있으므로 다음 원칙을 따른다.

- 종목 추천은 제공하지 않는다.
- 수익 보장을 암시하지 않는다.
- 사용자의 입력값을 바탕으로 한 일반적 가이드를 제공한다.
- 실제 상품 가입 전 조건과 위험 확인이 필요함을 명시한다.
- 고위험 운용 방식은 기본 추천 축이 아니라 제한적 보조 정보로만 다룬다.

## 13. 결론

본 서비스는 가계부의 기록 기능을 기반으로 사용자의 현금흐름을 분석하고, 이를 저축 운영 방식 추천으로 연결하는 서비스이다.  
초기 단계에서는 기능 확장보다 범위 고정과 추천 로직의 명확화가 중요하며, 사용자가 최소한의 입력만으로 실행 가능한 저축 계획을 수립할 수 있도록 설계해야 한다.

서비스의 차별점은 기록된 소비 데이터를 단순 통계로 끝내지 않고, 사용자의 재무 우선순위와 행동 계획으로 전환해준다는 점에 있다.
