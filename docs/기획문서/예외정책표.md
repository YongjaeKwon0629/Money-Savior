# 가계부 기반 저축 운영 추천 서비스 MVP 예외 정책 표

## 1. 문서 개요

본 문서는 MVP 단계에서 일반 추천 로직만으로 처리하기 어려운 예외 상황과 그에 대한 서비스 대응 원칙을 정의하기 위한 문서이다.  
예외 정책은 추천 실패를 선언하기 위한 것이 아니라, 사용자의 현재 재무 상황에 맞게 우선순위를 전환하고 보다 보수적인 안내를 제공하기 위한 목적을 가진다.

## 2. 예외 정책 기본 원칙

- 예외 상황에서는 공격적 추천을 제한하고 보수적 추천을 기본값으로 한다.
- 예외 상황에서는 추천 결과보다 경고 또는 우선 조치 안내를 상단에 노출한다.
- 예외 상황에서도 사용자가 다음 행동을 결정할 수 있도록 실행 가능한 안내를 제공한다.
- 예외 정책은 시스템 차단보다는 우선순위 재정렬 중심으로 설계한다.

| 예외 코드 | 기본 처리 정책 | 추천 결과 영향 | 사용자 메시지 방향 |
|---|---|---|---|
| EX-01 | 일반 추천 중단 후 지출 조정 우선형 적용 | 모든 저축액 0원 처리 가능 | 저축 확대보다 지출 구조 조정 우선 안내 |
| EX-02 | 공격형 추천 비활성화 | 적립식 운용 비중 제거 또는 축소 | 고정비 압박 완화 필요 안내 |
| EX-03 | 유동성 확보형 강제 적용 | 파킹통장 중심으로 전환 | 비상금 1순위 확보 안내 |
| EX-04 | 비상금 보완형으로 조정 | 투자형 비중 축소 | 비상금 목표 충족 전까지 보수적 운영 안내 |
| EX-05 | 목표 재설정 유도 | 도전 저축액 노출 제한 가능 | 단계적 목표 설정 필요 안내 |
| EX-06 | 성향보다 상황 우선 적용 | 안정형 또는 균형형 구조로 보정 | 현재 상황 기준 보수적 운영 안내 |
| EX-07 | 입력 재확인 또는 보수적 추천 적용 | 저축액 범위 하향 조정 | 수입 변동성 고려 필요 안내 |
| EX-08 | 추천 생성 제한 또는 입력 재요청 | 결과 신뢰도 하락 표시 | 정확한 입력 필요 안내 |

## 5. 세부 예외 정책

### 5.1 EX-01 월 잉여자금 부족

- 조건: 월 잉여자금 0원 이하
- 처리:
- 안전 저축액, 권장 저축액, 도전 저축액을 모두 0원으로 계산 가능
- 추천 구조는 `지출 조정 우선형`으로 고정
- 결과 화면에서 자금 배분보다 지출 구조 점검 항목을 우선 노출

권장 메시지:

- 현재는 신규 저축 확대보다 지출 구조 조정이 우선입니다.
- 이번 달은 고정비와 생활비를 먼저 점검한 후 저축 계획을 다시 설정하는 것이 적절합니다.

### 5.2 EX-02 고정비 과다

- 조건: 월 고정비 비중 50% 이상
- 처리:
- 적극형 또는 분산 운용형 추천 비활성화
- 적금 중심 또는 유동성 중심 구조 유지
- 고정비 항목 재점검 유도

권장 메시지:

- 고정비 비중이 높아 저축 여력이 제한되고 있습니다.
- 반복 지출을 먼저 조정하면 더 안정적인 저축 구조를 만들 수 있습니다.

### 5.3 EX-03 비상금 없음

- 조건: 현재 비상금 보유액 0원
- 처리:
- 추천 방식은 `유동성 확보형` 우선 적용
- ISA 및 일반 적립식 운용은 노출하지 않거나 참고 수준으로만 제한
- 파킹통장 중심 자금 배분 제안

권장 메시지:

- 현재는 비상 상황에 대응할 수 있는 자금이 없어 유동성 확보가 가장 중요합니다.
- 단기 운용보다 먼저 바로 쓸 수 있는 비상금을 마련하는 것이 적절합니다.

### 5.4 EX-04 비상금 부족

- 조건: 비상금 충족률 1% 이상 100% 미만
- 처리:
- 적극형 성향이어도 기본 추천은 균형 저축형 이하로 제한
- 파킹통장 또는 단기 안전 자산 비중 유지

권장 메시지:

- 비상금이 아직 목표 수준에 도달하지 않았습니다.
- 현재는 저축과 함께 유동성 확보를 병행하는 구조가 적절합니다.

### 5.5 EX-05 목표 과다

- 조건: 사용자 희망 저축 목표가 도전 저축액을 지속적으로 초과
- 처리:
- 사용자 목표와 시스템 권장값을 분리 노출
- 목표 달성까지 필요한 기간 재산정 가능

권장 메시지:

- 현재 목표 금액은 현금흐름 대비 다소 높은 수준입니다.
- 무리한 저축보다 단계적으로 목표를 나누어 설정하는 방식이 더 안정적입니다.

### 5.6 EX-06 성향-상황 불일치

- 조건: 적극형 성향이지만 잉여자금 낮음 또는 비상금 부족 상태
- 처리:
- 적극형 문구는 유지하되 실제 추천은 안정형 또는 균형형 구조로 보정
- 추천 이유에 보정 사유 명시

권장 메시지:

- 현재 성향은 적극형이지만, 현금흐름과 비상금 상태를 고려하면 보수적 운영이 더 적절합니다.
- 자금 여력이 안정되면 점진적으로 운용 비중을 확대할 수 있습니다.

### 5.7 EX-07 수입 불안정 가능성

- 조건: 수입 입력이 일관되지 않거나 부수입 의존도가 높음
- 처리:
- 권장 저축액 기준을 하향 조정
- 안전 저축액 중심으로 노출

권장 메시지:

- 수입 변동 가능성이 있어 보수적인 저축 기준을 적용했습니다.
- 일정 기간 수입 흐름을 확인한 뒤 계획을 조정하는 것이 적절합니다.

### 5.8 EX-08 입력값 신뢰도 낮음

- 조건: 필수값 누락, 음수 값 입력, 과도하게 비현실적인 값 입력
- 처리:
- 추천 생성 제한
- 입력 재확인 유도

권장 메시지:

- 입력값을 다시 확인하면 더 정확한 추천을 받을 수 있습니다.
- 현재 정보만으로는 신뢰 가능한 결과를 만들기 어렵습니다.

## 7. UI 반영 원칙

- 심각도 높음 예외는 결과 화면 상단 경고 영역에 노출한다.
- 심각도 중간 예외는 추천 이유 또는 주의 포인트 영역에 포함한다.
- 예외 메시지는 위협적 표현보다 우선순위 전환 안내 표현을 사용한다.
- 예외가 있더라도 사용자가 실행할 수 있는 다음 행동을 반드시 제공한다.

## 8. 결론

MVP 예외 정책은 추천 엔진의 한계를 감추기 위한 장치가 아니라, 사용자 상황에 맞는 안전장치를 제공하기 위한 핵심 요소이다.  
따라서 예외 상황은 최소한의 수로 정의하되, 실제 추천 구조를 바꾸는 명확한 기준과 사용자 안내 원칙을 함께 가져가야 한다.
