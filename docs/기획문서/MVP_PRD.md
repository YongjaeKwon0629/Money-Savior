# 가계부 기반 저축 운영 추천 서비스 MVP PRD

## 1. 문서 개요

본 문서는 가계부 기반 저축 운영 추천 서비스의 MVP 개발을 위한 제품 요구사항 문서이다.  
MVP의 목적은 사용자가 최소한의 입력만으로 자신의 월별 현금흐름을 파악하고, 이에 적합한 저축 운영 방식을 추천받을 수 있는 경험을 검증하는 데 있다.

본 PRD는 다음 문서를 기반으로 작성한다.

- 서비스기획서.md
- 추천로직표.md
- 예외정책표.md
- 결과화면정의서.md

## 2. 제품 목표

### 2.1 목표

- 사용자가 월별 현금흐름을 빠르게 입력할 수 있도록 한다.
- 사용자가 이번 달 저축 가능액을 이해할 수 있도록 한다.
- 사용자가 자신의 상황에 맞는 저축 운영 방식을 추천받을 수 있도록 한다.
- 사용자가 추천 결과를 바탕으로 이번 달 실행 계획을 세울 수 있도록 한다.

### 2.2 비목표

- 개별 종목 추천 제공
- 자동 투자 실행
- 실시간 금융상품 비교
- 복잡한 포트폴리오 분석
- 종합 재무설계 기능 제공

## 3. MVP 범위

### 3.1 포함 범위

- 기본 정보 입력
- 월별 지출 정보 입력
- 저축 성향 및 저축 목표 입력
- 월 잉여자금 계산
- 비상금 상태 계산
- 저축 가능액 계산
- 저축 운영 방식 추천
- 결과 화면 제공

### 3.2 제외 범위

- 자동 계좌 연동
- 금융사 API 연동
- 상세 카테고리 소비 분석
- 월별 리포트 히스토리
- 자산 포트폴리오 관리
- 고도화된 시뮬레이션

## 4. 대상 사용자

MVP의 주요 대상 사용자는 다음과 같다.

- 고정적인 월급을 수령하는 직장인
- 자산관리를 처음 시작하거나 기준이 필요한 사용자
- 가계부를 기록해도 저축 방식까지 연결하지 못한 사용자

## 5. 사용자 핵심 문제

- 이번 달 실제로 얼마를 저축할 수 있는지 알기 어렵다.
- 남는 자금을 어떤 방식으로 배분해야 하는지 판단하기 어렵다.
- 비상금, 적금, ISA, 적립식 운용 중 무엇이 우선인지 모르겠다.
- 소비 기록은 하지만 행동 가능한 계획으로 이어지지 않는다.

## 6. 핵심 가설

- 사용자는 복잡한 금융상품 비교보다 자신의 상황에 맞는 저축 운영 가이드를 더 필요로 한다.
- 사용자는 3분 내외의 간단한 입력만으로도 가치 있는 추천을 받는다고 느낄 수 있다.
- 사용자는 추천 결과에 숫자와 이유가 함께 제공될 때 더 높은 신뢰를 느낀다.

## 7. 필수 기능 요구사항

### 7.1 온보딩 및 입력

#### 기능 설명

사용자로부터 추천 생성에 필요한 최소 정보를 입력받는다.

#### 입력 항목

- 월 실수령액
- 월급일
- 월 고정비
- 월 변동지출
- 현재 비상금 보유액
- 저축 목표
- 저축 성향

#### 요구사항

- 모든 금액은 월 기준으로 입력받아야 한다.
- 수입은 세후 실수령액 기준임을 안내해야 한다.
- 월 고정비와 월 변동지출은 총액 입력을 기본으로 제공해야 한다.
- 필수 항목 누락 시 결과 생성이 불가능해야 한다.

### 7.2 계산 엔진

#### 기능 설명

입력값을 기반으로 핵심 재무 상태를 계산한다.

#### 필수 계산 항목

- 월 잉여자금
- 고정비 비중
- 생활비 기준액
- 비상금 목표액
- 비상금 충족률
- 안전 저축액
- 권장 저축액
- 도전 저축액

#### 요구사항

- 월 잉여자금은 월 실수령액 - 월 고정비 - 월 변동지출로 계산해야 한다.
- 비상금 목표액은 생활비 기준액 x 3개월로 계산해야 한다.
- 잉여자금이 0원 이하일 경우 저축 가능액은 0원으로 처리 가능해야 한다.

### 7.3 추천 엔진

#### 기능 설명

계산 결과와 사용자 입력을 기반으로 저축 운영 방식을 추천한다.

#### 추천 방식 유형

- 지출 조정 우선형
- 유동성 확보형
- 안정 저축형
- 균형 저축형
- 분산 운용형

#### 요구사항

- 추천 단위는 종목이 아니라 운영 방식이어야 한다.
- 추천 로직은 저축 여력, 비상금 상태, 저축 성향을 기본 판단 기준으로 사용해야 한다.
- 목표에 따라 자금 배분 구조를 보정할 수 있어야 한다.
- 추천 결과에는 추천 이유를 최소 2개 이상 포함해야 한다.

### 7.4 예외 처리

#### 기능 설명

일반 추천으로 처리하기 어려운 상황을 별도 정책에 따라 보정한다.

#### 주요 예외

- 월 잉여자금 부족
- 고정비 과다
- 비상금 없음
- 비상금 부족
- 목표 과다
- 성향-상황 불일치
- 입력값 신뢰도 낮음

#### 요구사항

- 예외 상황 발생 시 보수적 추천을 우선 적용해야 한다.
- 심각도 높은 예외는 결과 화면 상단에 우선 노출해야 한다.
- 예외 메시지는 경고보다 우선 조치 안내 중심으로 작성해야 한다.

### 7.5 결과 화면

#### 기능 설명

사용자에게 추천 결과와 실행 가능한 계획을 제공한다.

#### 필수 노출 정보

- 이번 달 현금흐름 요약
- 비상금 상태
- 안전/권장/도전 저축액
- 추천 운영 방식
- 자금 배분 구조
- 추천 이유
- 주의 포인트
- 실행 액션

#### 요구사항

- 권장 저축액을 가장 강조된 형태로 노출해야 한다.
- 예외 상황 존재 시 경고 또는 우선 조치 메시지를 상단에 노출해야 한다.
- 실행 액션은 최소 1개 이상 제공해야 한다.

## 8. 비기능 요구사항

- 사용자는 전체 입력을 3분 이내에 완료할 수 있어야 한다.
- 결과는 입력 완료 직후 즉시 생성되어야 한다.
- 문구는 금융 권유 표현보다 일반적 가이드 표현을 사용해야 한다.
- 모바일 환경에서도 핵심 결과를 한 화면 내에서 이해할 수 있어야 한다.

## 9. 성공 지표

### 9.1 제품 지표

- 입력 완료율
- 결과 화면 도달율
- 추천 계획 저장 또는 실행 액션 클릭율

### 9.2 검증 지표

- 사용자가 결과를 이해했다고 응답하는 비율
- 추천이 현실적이라고 응답하는 비율
- 입력 과정이 과도하게 번거롭지 않다고 응답하는 비율

## 10. 사용자 흐름

1. 사용자가 서비스에 진입한다.
2. 월 실수령액과 월급일을 입력한다.
3. 월 고정비와 월 변동지출을 입력한다.
4. 현재 비상금 보유액을 입력한다.
5. 저축 목표와 저축 성향을 선택한다.
6. 시스템이 계산 및 추천 로직을 수행한다.
7. 시스템이 결과 화면을 제공한다.
8. 사용자가 추천 계획을 확인하고 저장 또는 다음 행동을 선택한다.

## 11. 화면 구성

MVP에서 필요한 화면은 다음과 같다.

- 온보딩 화면
- 기본 정보 입력 화면
- 지출 입력 화면
- 목표 및 성향 입력 화면
- 결과 화면

## 12. 정책 및 제약 사항

- 본 서비스는 종목 추천을 제공하지 않는다.
- 본 서비스는 수익률 보장을 암시하지 않는다.
- 본 서비스는 입력값 기반 일반적 가이드만 제공한다.
- 실제 금융상품 가입 전 조건과 위험은 사용자가 별도 확인해야 한다.

## 13. 향후 확장 항목

- 월별 결과 히스토리 저장
- 지출 카테고리 상세 분석
- 목표 기간 및 투자 경험 기반 세부 보정
- 대시보드 및 월별 리포트
- 장기 자산관리 코치 기능

## 14. 오픈 이슈

- 저축 여력 단계 기준 수치의 세부 조정 필요
- 목표 유형 분류 체계 세분화 필요
- 결과 화면에서 금액과 비율 중 어떤 표현을 기본으로 할지 결정 필요
- 계획 저장 기능의 MVP 포함 범위 결정 필요

## 15. 결론

본 MVP의 핵심은 복잡한 재무설계 서비스 구현이 아니라, 사용자가 최소한의 입력으로 자신의 현금흐름을 이해하고 저축 운영 방향을 결정할 수 있게 만드는 것이다.  
초기 단계에서는 입력 간결성, 추천 일관성, 결과 이해 가능성을 우선 검증해야 하며, 이후 실제 사용자 반응을 바탕으로 로직과 기능을 점진적으로 고도화한다.
