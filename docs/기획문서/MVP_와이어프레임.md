# 가계부 기반 저축 운영 추천 서비스 MVP 와이어프레임

## 1. 문서 개요

본 문서는 가계부 기반 저축 운영 추천 서비스의 MVP 화면 구조를 정의하기 위한 와이어프레임 문서이다.  
본 문서의 목적은 서비스 기획서, 추천 로직, 예외 정책, 결과 화면 정의를 실제 화면 흐름과 UI 구성으로 연결하는 데 있다.

본 문서의 범위는 다음과 같다.

- MVP 화면 흐름 정의
- 화면별 목적 및 핵심 정보 구조 정의
- 화면별 CTA 정의
- 예외 상황 반영 방식 정의

## 2. 와이어프레임 설계 원칙

- 사용자는 3분 이내에 입력을 완료할 수 있어야 한다.
- 한 화면에는 한 가지 핵심 입력 또는 핵심 판단만 담는다.
- 입력보다 결과 해석이 더 중요하므로 결과 화면에 가장 많은 정보 밀도를 배치한다.
- 모바일 기준 세로 흐름을 기본으로 설계한다.
- 화면 전환마다 사용자가 현재 진행 상태를 인지할 수 있어야 한다.

## 3. MVP 전체 화면 흐름

MVP 화면 흐름은 아래 순서를 따른다.

1. 온보딩 화면
2. 기본 정보 입력 화면
3. 지출 정보 입력 화면
4. 목표 및 성향 입력 화면
5. 결과 화면

선택적으로 아래 보조 화면을 둘 수 있다.

- 입력값 수정 화면
- 계획 저장 완료 토스트 또는 모달

## 4. 정보 구조 개요

| 화면 | 주요 목적 | 핵심 정보 | 주요 CTA |
|---|---|---|---|
| 온보딩 | 서비스 가치 전달 및 시작 유도 | 서비스 설명, 기대 결과 | 시작하기 |
| 기본 정보 입력 | 수입과 급여 기준 수집 | 월 실수령액, 월급일, 비상금 | 다음 |
| 지출 정보 입력 | 월 지출 구조 수집 | 월 고정비, 월 변동지출 | 다음 |
| 목표 및 성향 입력 | 추천 기준 보강 | 저축 목표, 저축 성향 | 결과 보기 |
| 결과 화면 | 추천 결과 해석 및 실행 유도 | 권장 저축액, 추천 방식, 이유, 액션 | 계획 저장, 다시 진단 |

## 5. 화면별 와이어프레임

## 5.1 온보딩 화면

### 화면 목적

- 서비스가 무엇을 해주는지 짧고 명확하게 전달한다.
- 사용자가 입력 과정에 대한 부담 없이 진입할 수 있도록 유도한다.

### 핵심 메시지

- 이번 달 얼마를 저축할 수 있는지 계산합니다.
- 내 상황에 맞는 저축 운영 방식을 추천합니다.
- 복잡한 투자 추천이 아니라 실행 가능한 저축 플랜을 제공합니다.

### 레이아웃 구조

```text
[상단 로고 또는 서비스명]

[메인 헤드라인]
내 월급 기준으로
어떻게 저축하면 좋을지 알려드립니다

[서브 설명]
수입, 지출, 비상금 정보를 입력하면
이번 달 저축 가능액과 추천 운영 방식을 안내합니다

[핵심 가치 카드 3개]
- 현금흐름 분석
- 저축 가능액 계산
- 저축 방식 추천

[하단 고정 버튼]
시작하기
```

### 주요 UI 요소

- 메인 타이틀
- 서브 카피
- 핵심 가치 카드
- 시작 버튼

### CTA

- `시작하기`

### 예외 처리

- 없음

---

## 5.2 기본 정보 입력 화면

### 화면 목적

- 추천 계산에 필요한 최소한의 기본 입력값을 수집한다.

### 수집 항목

- 월 실수령액
- 월급일
- 현재 비상금 보유액

### 레이아웃 구조

```text
[상단 진행 상태]
1 / 4

[화면 제목]
기본 정보를 입력해주세요

[설명]
세후 월급과 현재 보유한 비상금 기준으로 계산합니다

[입력 필드]
월 실수령액
[__________] 원

월급일
[매월 25일 v]

현재 비상금 보유액
[__________] 원

[하단 버튼]
다음
```

### 주요 UI 요소

- 진행 단계 표시
- 숫자 입력 필드
- 월급일 선택 드롭다운 또는 피커
- 다음 버튼

### UX 메모

- 월 실수령액 입력 아래 `세후 기준` 안내 문구 노출
- 비상금 입력 아래 `예금, 파킹통장 등 바로 쓸 수 있는 현금성 자산 기준` 보조 문구 노출
- 필수값 누락 시 버튼 비활성화

### CTA

- `다음`

### 검증 규칙

- 월 실수령액은 0보다 커야 한다.
- 월급일은 1일부터 31일까지 선택 가능해야 한다.
- 비상금 보유액은 0 이상이어야 한다.

### 예외 처리

- 비현실적인 금액 입력 시 인라인 에러 노출
- 필수값 미입력 시 다음 버튼 비활성화

## 5.3 지출 정보 입력 화면

### 화면 목적

- 월간 지출 구조를 간단하게 수집한다.

### 수집 항목

- 월 고정비
- 월 변동지출

### 레이아웃 구조

```text
[상단 진행 상태]
2 / 4

[화면 제목]
매달 나가는 돈을 입력해주세요

[설명]
상세 항목 대신 월 총액만 입력해도 괜찮습니다

[입력 필드]
월 고정비
[__________] 원

월 변동지출
[__________] 원

[보조 안내]
고정비: 월세, 통신비, 보험, 구독료
변동지출: 식비, 교통비, 쇼핑, 취미

[하단 버튼]
이전   다음
```

### 주요 UI 요소

- 금액 입력 필드 2개
- 입력 예시 문구
- 이전/다음 버튼

### UX 메모

- 사용자가 세부 카테고리를 몰라도 입력 가능하도록 총액 중심 구성
- 필드 하단에 대표 예시를 짧게 노출

### CTA

- `이전`
- `다음`

### 검증 규칙

- 월 고정비는 0 이상이어야 한다.
- 월 변동지출은 0 이상이어야 한다.
- 월 고정비 + 월 변동지출이 월 실수령액보다 과도하게 큰 경우 경고 메시지 노출 가능

### 예외 처리

- 입력값이 과도하게 높을 경우 `다시 확인해보세요` 인라인 메시지 노출

## 5.4 목표 및 성향 입력 화면

### 화면 목적

- 추천 로직에 필요한 목표와 성향을 수집한다.

### 수집 항목

- 저축 목표
- 저축 성향

### 레이아웃 구조

```text
[상단 진행 상태]
3 / 4

[화면 제목]
저축 목표와 성향을 알려주세요

[질문 1]
무엇을 위해 저축하고 싶나요?
[비상금]
[생활 안정]
[여행]
[주거/결혼]
[투자 준비]

[질문 2]
어떤 방식이 편한가요?
[안정형]
원금 변동이 적은 방식 선호

[균형형]
안정성과 성장 가능성을 함께 고려

[적극형]
장기적으로 운용 비중 확대 가능

[하단 버튼]
이전   결과 보기
```

### 주요 UI 요소

- 목표 선택 칩 또는 카드
- 성향 선택 카드
- 결과 보기 버튼

### UX 메모

- 목표는 단일 선택으로 시작하는 것이 MVP에 적합
- 성향 설명 문구는 짧고 직관적으로 유지

### CTA

- `이전`
- `결과 보기`

### 검증 규칙

- 목표 1개 이상 선택 필수
- 성향 1개 선택 필수

### 예외 처리

- 미선택 시 결과 보기 버튼 비활성화

## 5.5 결과 화면

### 화면 목적

- 계산 결과와 추천 내용을 전달하고, 사용자가 바로 행동할 수 있도록 만든다.

### 레이아웃 구조

```text
[상단 헤더]
이번 달 저축 플랜

[예외 경고 배너 - 조건부]
현재는 저축 확대보다 지출 구조 조정이 우선입니다

[상태 요약 카드]
월 실수령액 250만 원
월 고정비 90만 원
월 변동지출 110만 원
월 잉여자금 50만 원
비상금 상태 부족

[권장 저축액 영역]
안전 25만 원
권장 35만 원
도전 45만 원

[추천 운영 방식 카드]
추천 방식: 균형 저축형
파킹통장 10만 원
적금 15만 원
ISA 적립 10만 원

[추천 이유 영역]
- 잉여자금이 안정적으로 확보되어 있습니다
- 비상금이 아직 충분하지 않습니다
- 균형형 성향을 반영해 안정 저축과 적립식 운용을 함께 제안합니다

[주의 포인트 영역 - 조건부]
- 고정비 비중이 다소 높습니다
- 비상금이 목표 수준에 도달하지 않았습니다

[실행 액션 영역]
[추천 계획 저장]
[다시 진단하기]
```

### 주요 UI 요소

- 결과 헤더
- 예외 경고 배너
- 상태 요약 카드
- 저축 가능액 3단 비교 카드
- 추천 운영 방식 카드
- 이유 리스트
- 주의 포인트 리스트
- 실행 버튼

### 정보 우선순위

1. 예외 경고 또는 상태 요약
2. 권장 저축액
3. 추천 운영 방식
4. 추천 이유
5. 주의 포인트
6. 실행 액션

### CTA

- `추천 계획 저장`
- `다시 진단하기`

### 예외 케이스 와이어프레임

```text
[상단 헤더]
이번 달 저축 플랜

[강조 경고 배너]
현재는 신규 저축보다 지출 구조 조정이 우선입니다

[상태 요약 카드]
월 잉여자금 0원
비상금 상태 없음

[권장 저축액 영역]
안전 0원
권장 0원
도전 0원

[추천 운영 방식 카드]
추천 방식: 지출 조정 우선형

[우선 조치 영역]
- 고정비 재점검
- 생활비 감축 계획 세우기
- 비상금 1차 목표 먼저 설정하기

[실행 액션]
[입력값 수정]
[다시 진단하기]
```

## 6. 상태별 UI 정책

| 상태 | 처리 방식 |
|---|---|
| 정상 입력 | 일반 결과 화면 노출 |
| 경미한 주의 필요 | 결과 화면 내 주의 포인트 영역 노출 |
| 심각한 예외 발생 | 결과 상단 경고 배너 우선 노출 |
| 입력 오류 | 해당 입력 화면 인라인 에러 처리 |

## 7. 공통 UI 컴포넌트 제안

- 진행 상태 바
- 숫자 입력 필드
- 선택형 카드
- 요약 정보 카드
- 경고 배너
- 하단 고정 CTA 버튼

## 8. 모바일 우선 설계 메모

- 모든 화면은 세로 스크롤 기준 단일 컬럼 구조로 설계한다.
- CTA 버튼은 하단 고정 또는 마지막 영역에 명확히 배치한다.
- 숫자 입력은 키패드 친화적으로 구성한다.
- 결과 화면은 핵심 숫자가 첫 화면 내에 보이도록 상단 압축 배치가 필요하다.

## 9. 향후 확장 포인트

- 결과 화면 내 월별 비교 카드 추가
- 입력 화면에서 상세 지출 카테고리 확장
- 결과 화면에서 추천 계획 수정 기능 추가
- 대시보드형 홈 화면 추가

## 10. 결론

본 MVP 와이어프레임은 입력 부담을 최소화하면서도 결과 화면의 설득력을 최대화하는 구조를 목표로 한다.  
초기 단계에서는 화면 수를 늘리기보다, 입력 흐름의 자연스러움과 결과 화면의 이해 가능성을 우선 검증하는 것이 중요하다.
