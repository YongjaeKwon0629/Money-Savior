# 가계부 기반 저축 운영 추천 서비스 MVP 결과 화면 정의서

## 1. 문서 개요

본 문서는 MVP 단계에서 제공할 결과 화면의 목적, 구성 요소, 노출 우선순위, 사용자 액션을 정의하기 위한 문서이다.  
결과 화면은 단순히 계산 결과를 보여주는 화면이 아니라, 사용자가 이번 달 저축 계획을 이해하고 실행할 수 있도록 돕는 핵심 화면으로 정의한다.

## 2. 화면 목표

결과 화면의 목표는 다음과 같다.

- 사용자가 자신의 월별 현금흐름 상태를 빠르게 이해할 수 있도록 한다.
- 사용자가 이번 달 저축 가능액과 추천 운영 방식을 즉시 파악할 수 있도록 한다.
- 사용자가 추천의 근거를 이해하고 신뢰할 수 있도록 한다.
- 사용자가 다음 행동을 선택할 수 있도록 실행 액션을 제공한다.

## 3. 화면 핵심 원칙

- 가장 중요한 정보는 상단에 우선 배치한다.
- 결과는 숫자와 설명을 함께 제공한다.
- 추천 이유를 반드시 포함하여 결과 해석을 돕는다.
- 예외 상황이 있는 경우 추천보다 경고 또는 우선 조치를 먼저 보여준다.
- 사용자가 한 번에 이해할 수 있도록 정보 밀도를 제한한다.

## 4. 화면 구성 구조

결과 화면은 아래 영역으로 구성한다.

1. 상태 요약 영역
2. 저축 가능액 영역
3. 추천 운영 방식 영역
4. 추천 이유 영역
5. 주의 포인트 영역
6. 실행 액션 영역

## 5. 상세 영역 정의

### 5.1 상태 요약 영역

목적:

- 사용자의 이번 달 재무 상태를 한눈에 보여준다.

노출 정보:

- 월 실수령액
- 월 고정비
- 월 변동지출
- 월 잉여자금
- 비상금 상태 요약

표현 방식:

- 숫자 중심 카드 또는 요약 박스
- 비상금 상태는 `없음`, `부족`, `충족` 형태의 태그 또는 배지로 표현

### 5.2 저축 가능액 영역

목적:

- 사용자가 이번 달에 감당 가능한 저축 범위를 이해할 수 있도록 한다.

노출 정보:

- 안전 저축액
- 권장 저축액
- 도전 저축액

표현 방식:

- 3단계 비교 카드
- 기본 강조는 권장 저축액

정책:

- 잉여자금이 0원 이하일 경우 세 금액 모두 0원으로 표시 가능
- 예외 상황이 존재할 경우 권장 저축액 산출 기준을 함께 안내

### 5.3 추천 운영 방식 영역

목적:

- 사용자가 자금을 어떤 방식으로 나누어 운영해야 하는지 바로 이해할 수 있도록 한다.

노출 정보:

- 추천 방식명
- 자금 배분 구조
- 배분 비율 또는 금액

예시:

- 추천 방식: 균형 저축형
- 추천 구조: 파킹통장 10만 원 / 적금 20만 원 / ISA 적립 15만 원

정책:

- 추천은 하나의 대표 방식으로 제시하되, 세부 배분은 함께 제공한다.
- 고위험 자산 운용은 MVP 기본 구조에서 직접 강조하지 않는다.

### 5.4 추천 이유 영역

목적:

- 사용자가 왜 이런 추천을 받았는지 이해할 수 있도록 한다.

노출 정보:

- 추천 판단 근거 2~3개

예시:

- 현재 잉여자금이 안정적으로 확보되어 있습니다.
- 비상금이 아직 충분하지 않아 유동성 자금 확보가 필요합니다.
- 균형형 성향을 반영해 안정 저축과 적립식 운용을 병행하는 구조를 제안합니다.

정책:

- 숫자 해석, 비상금 상태, 성향 반영 여부를 중심으로 설명한다.
- 문장은 짧고 명확하게 유지한다.

### 5.5 주의 포인트 영역

목적:

- 추천을 그대로 따르기 전에 알아야 할 주의사항을 전달한다.

노출 정보:

- 비상금 부족 여부
- 고정비 압박 여부
- 목표 과다 여부
- 입력값 제한에 따른 해석 유의사항

정책:

- 예외 상황이 없는 경우 간략 노출 또는 생략 가능
- 예외 상황이 있는 경우 결과 화면 상단 또는 추천 영역 직하단에 우선 노출

### 5.6 실행 액션 영역

목적:

- 사용자가 결과를 본 후 바로 행동으로 옮길 수 있도록 한다.

가능 액션:

- 추천 계획 저장
- 이번 달 목표 설정
- 월급일 기준 자동이체 일정 메모
- 다음 달 다시 진단하기

정책:

- MVP에서는 실제 자동이체 실행 기능 대신 계획 저장 또는 메모 수준의 액션으로 제한 가능

## 6. 정보 우선순위

결과 화면의 정보 노출 순서는 아래와 같다.

1. 예외 경고 또는 상태 요약
2. 권장 저축액
3. 추천 운영 방식
4. 추천 이유
5. 주의 포인트
6. 실행 액션

## 7. 예외 상황 반영 기준

### 7.1 예외 상황 없음

- 상태 요약과 권장 저축액을 최상단에 배치
- 추천 운영 방식과 추천 이유를 자연스럽게 이어서 제공

### 7.2 심각도 높은 예외 존재

- 상단 경고 배너 우선 노출
- 추천 운영 방식보다 주의 메시지와 우선 조치 안내를 먼저 제공
- 실행 액션도 `저축 시작`보다 `지출 점검` 또는 `비상금 확보` 중심으로 조정

## 8. 결과 화면 예시 구조

### 8.1 일반 케이스

- 이번 달 현금흐름 요약
- 권장 저축액 45만 원
- 추천 운영 방식: 균형 저축형
- 자금 배분: 파킹통장 10만 원 / 적금 20만 원 / ISA 적립 15만 원
- 추천 이유 3가지
- 이번 달 실행 액션

### 8.2 예외 케이스

- 경고: 현재는 저축 확대보다 지출 구조 조정이 우선입니다.
- 이번 달 현금흐름 요약
- 권장 저축액 0원
- 추천 운영 방식: 지출 조정 우선형
- 우선 조치: 고정비 재점검, 생활비 감축 계획 수립

## 9. MVP 범위 및 비포함 항목

### 9.1 MVP 포함

- 단일 결과 화면
- 핵심 숫자 요약
- 추천 운영 방식 1종
- 추천 이유
- 주의 포인트
- 실행 액션

### 9.2 MVP 비포함

- 복수 시나리오 비교
- 월별 추이 그래프
- 금융상품 상세 비교
- 실시간 시장 정보 반영
- 장기 포트폴리오 시뮬레이션

## 10. 결론

결과 화면은 서비스의 핵심 가치를 전달하는 중심 화면이다.  
따라서 단순한 분석 결과 나열보다, 사용자가 자신의 상황을 이해하고 이번 달 행동을 결정할 수 있도록 만드는 구조를 우선해야 한다.  
MVP에서는 정보의 양보다 실행 가능성과 이해 가능성을 우선하는 설계가 필요하다.
