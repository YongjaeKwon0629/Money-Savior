# 가계부 기반 저축 운영 추천 서비스 MVP 로우파이 UI 상세안

## 1. 문서 개요

본 문서는 MVP 와이어프레임을 기반으로 실제 화면 설계에 바로 활용할 수 있는 로우파이 UI 상세안을 정의하기 위한 문서이다.  
본 문서는 시각 디자인보다 정보 배치, 컴포넌트 구조, 상태 변화, 사용자 행동 흐름을 명확히 하는 데 목적이 있다.

## 2. 문서 활용 목적

- 디자인 시안 제작 전 화면 구조를 고정한다.
- 프론트엔드 구현 전 필요한 UI 요소를 명확히 정의한다.
- 화면별 우선순위와 상태별 반응을 통일한다.

## 3. 공통 설계 원칙

- 모바일 우선 단일 컬럼 레이아웃을 기본으로 한다.
- 화면 상단에는 현재 단계 또는 현재 상태를 인지할 수 있는 정보가 있어야 한다.
- 입력 화면은 한 번에 하나의 인지 과제만 주도록 구성한다.
- 결과 화면은 첫 화면 내에서 핵심 결과가 보이도록 상단 압축 배치를 적용한다.
- CTA는 매 화면 하단에서 명확히 구분되어야 한다.

## 4. 공통 레이아웃 구조

모든 주요 화면은 아래 구조를 따른다.

```text
[상단 안전 영역]
[헤더 또는 진행 상태]
[본문 콘텐츠]
[하단 고정 CTA 영역]
```

### 4.1 공통 컴포넌트

- `TopHeader`
- `ProgressIndicator`
- `SectionTitle`
- `HelperText`
- `NumberInputField`
- `SelectCard`
- `StatusBadge`
- `SummaryCard`
- `WarningBanner`
- `PrimaryButton`
- `SecondaryButton`

## 5.1 온보딩 화면 상세안

### 화면 목적

- 사용자가 서비스 가치를 5초 내에 이해하도록 만든다.
- 입력 시작 전 기대 결과를 명확히 전달한다.

### 화면 구조

```text
[TopHeader]
좌측: 로고 또는 서비스명
우측: 없음

[Hero Section]
메인 헤드라인
서브 설명

[Value Card Group]
가치 카드 3개 세로 배치

[Bottom CTA]
PrimaryButton: 시작하기
```

### 섹션 구성

#### A. Hero Section

- 헤드라인:
`이번 달 얼마를 저축할 수 있는지 바로 확인하세요`
- 서브카피:
`월급, 지출, 비상금 정보를 입력하면 지금 내 상황에 맞는 저축 운영 방식을 추천합니다`

#### B. Value Card Group

카드 1
- 타이틀: `현금흐름 분석`
- 설명: `수입과 지출을 기준으로 실제 여유 자금을 계산합니다`

카드 2
- 타이틀: `저축 가능액 제안`
- 설명: `안전, 권장, 도전 기준으로 저축 범위를 보여줍니다`

카드 3
- 타이틀: `저축 방식 추천`
- 설명: `파킹통장, 적금, ISA 적립 등 운영 방식을 제안합니다`

### UI 메모

- 첫 화면은 설명보다 행동 유도가 우선이다.
- 스크롤 없이 CTA가 보이는 구성이 이상적이다.

## 5.2 기본 정보 입력 화면 상세안

### 화면 목적

- 계산 엔진의 기준이 되는 핵심 정보 입력

### 화면 구조

```text
[TopHeader]
좌측: 뒤로가기
중앙: 기본 정보

[ProgressIndicator]
1 / 4

[SectionTitle]
기본 정보를 입력해주세요

[Field Group]
월 실수령액
월급일
현재 비상금 보유액

[Bottom CTA]
PrimaryButton: 다음
```

### 컴포넌트 구조

#### A. 월 실수령액 필드

- 라벨: `월 실수령액`
- 타입: 숫자 입력
- 접미 텍스트: `원`
- 플레이스홀더: `예: 2500000`
- 도움말: `세후 기준으로 입력해주세요`

상태
- 기본 상태
- 입력 중 상태
- 에러 상태: `0원보다 큰 금액을 입력해주세요`

#### B. 월급일 필드

- 라벨: `월급일`
- 타입: 드롭다운 또는 바텀시트 선택
- 기본값 없음
- 옵션 예시: `매월 5일`, `매월 10일`, `매월 25일`, `직접 선택`

#### C. 비상금 보유액 필드

- 라벨: `현재 비상금 보유액`
- 타입: 숫자 입력
- 접미 텍스트: `원`
- 도움말: `예금, 파킹통장 등 바로 사용할 수 있는 현금성 자산 기준`

### 하단 CTA 정책

- 필수값이 모두 유효할 때만 `다음` 버튼 활성화
- 키보드가 올라와도 버튼 접근이 가능해야 함

### 추가 UX 메모

- 사용자가 비상금 개념을 어려워할 수 있으므로 예시 설명이 중요함
- 월 실수령액 입력 시 천 단위 구분 표시 적용 권장

## 5.3 지출 정보 입력 화면 상세안

### 화면 목적

- 월간 지출 규모를 빠르게 파악

### 화면 구조

```text
[TopHeader]
좌측: 뒤로가기
중앙: 지출 정보

[ProgressIndicator]
2 / 4

[SectionTitle]
매달 나가는 돈을 입력해주세요

[HelperText]
정확한 금액이 아니어도 괜찮습니다

[Field Group]
월 고정비
월 변동지출

[Info Box]
고정비/변동지출 예시 설명

[Bottom CTA]
SecondaryButton: 이전
PrimaryButton: 다음
```

### 컴포넌트 구조

#### A. 월 고정비 필드

- 라벨: `월 고정비`
- 설명: `월세, 통신비, 보험, 구독료 등`
- 타입: 숫자 입력

#### B. 월 변동지출 필드

- 라벨: `월 변동지출`
- 설명: `식비, 교통비, 쇼핑, 취미 등`
- 타입: 숫자 입력

#### C. 안내 박스

- 타이틀: `대략 입력해도 됩니다`
- 내용:
`세부 항목을 모두 나누지 않아도 월 총액만 입력하면 추천을 받을 수 있습니다`

### 상태 정책

- 합산 지출이 수입보다 높은 경우 즉시 차단하지는 않음
- 대신 인라인 메시지 노출:
`현재 입력값 기준으로는 잉여자금이 없을 수 있습니다`

### 추가 UX 메모

- 이 화면은 사용자가 가장 이탈하기 쉬운 구간이므로 설명 밀도를 낮춰야 한다.
- 숫자 2개만 빠르게 입력하는 구조가 중요하다.

## 5.4 목표 및 성향 입력 화면 상세안

### 화면 목적

- 추천 해석에 필요한 사용자 의도와 성향 수집

### 화면 구조

```text
[TopHeader]
좌측: 뒤로가기
중앙: 목표와 성향

[ProgressIndicator]
3 / 4

[SectionTitle]
저축 목표와 성향을 알려주세요

[Question Block]
무엇을 위해 저축하고 싶나요?

[Goal Select Chips]
비상금 / 생활 안정 / 여행 / 주거·결혼 / 투자 준비

[Question Block]
어떤 방식이 편한가요?

[Preference Card Group]
안정형
균형형
적극형

[Bottom CTA]
SecondaryButton: 이전
PrimaryButton: 결과 보기
```

### 컴포넌트 구조

#### A. 목표 선택 칩

- 단일 선택
- 선택 시 배경 강조
- 선택 해제 가능 여부는 비허용이 자연스러움

#### B. 성향 선택 카드

카드 1
- 타이틀: `안정형`
- 설명: `원금 변동이 적은 방식이 편합니다`

카드 2
- 타이틀: `균형형`
- 설명: `안정성과 성장 가능성을 함께 보고 싶습니다`

카드 3
- 타이틀: `적극형`
- 설명: `장기적으로 운용 비중을 높일 수 있습니다`

### 상태 정책

- 목표와 성향 모두 선택해야 CTA 활성화
- 성향 설명은 금융 전문 용어 없이 유지

### 추가 UX 메모

- 이 화면은 선택형 인터랙션 위주로 구성해 입력 피로를 낮춘다.

## 5.5 결과 화면 상세안

### 화면 목적

- 이번 달 저축 계획을 이해시키고 바로 행동하게 만든다.

### 핵심 목표

- 3초 안에 권장 저축액 인지
- 10초 안에 추천 방식 이해
- 20초 안에 왜 그런 추천인지 납득

### 화면 구조

```text
[TopHeader]
좌측: 뒤로가기 또는 닫기
중앙: 이번 달 저축 플랜
우측: 공유 없음

[WarningBanner - 조건부]

[Hero Summary Card]
권장 저축액
비상금 상태
월 잉여자금

[Amount Tier Section]
안전 / 권장 / 도전

[Recommendation Card]
추천 방식명
배분 금액
배분 설명

[Reason Section]
추천 이유 리스트

[Caution Section - 조건부]
주의 포인트 리스트

[Action Section]
계획 저장
다시 진단하기
```

### 섹션별 상세 정의

#### A. WarningBanner

노출 조건:

- 심각도 높은 예외 존재 시

예시 문구:

- `현재는 저축 확대보다 지출 구조 조정이 우선입니다`
- `비상금이 없어 유동성 확보를 먼저 권장합니다`

UI 메모:

- 본문보다 먼저 보여야 함
- 빨간 경고보다는 주황 또는 중립 톤이 적합

#### B. Hero Summary Card

목적:

- 가장 중요한 결과만 압축해 첫 화면 안에 배치

노출 요소:

- 메인 라벨: `권장 저축액`
- 메인 값: `35만 원`
- 보조 정보 1: `월 잉여자금 50만 원`
- 보조 정보 2: `비상금 상태 부족`

UI 메모:

- 카드 하나로 결과 화면의 핵심을 전달해야 함
- 권장 저축액은 가장 큰 타이포 사용

#### C. Amount Tier Section

구성:

- 카드 3개 가로 또는 세로 배치

카드 구조:
- 라벨
- 금액
- 짧은 설명

예시:

- 안전: `25만 원`
  설명: `무리 없이 유지하기 쉬운 수준`
- 권장: `35만 원`
  설명: `현재 상황에서 가장 적절한 수준`
- 도전: `45만 원`
  설명: `지출 관리가 잘 될 때 가능한 수준`

#### D. Recommendation Card

목적:

- 사용자가 실제로 어떻게 돈을 나눌지 이해

카드 구조:

- 상단 태그: `추천 방식`
- 방식명: `균형 저축형`
- 배분 리스트
- 짧은 요약 설명

배분 리스트 예시:

- 파킹통장 10만 원
- 적금 15만 원
- ISA 적립 10만 원

요약 설명 예시:

- `비상금 보완과 저축 누적을 함께 가져가는 구조입니다`

#### E. Reason Section

구성:

- 제목: `이렇게 추천한 이유`
- 불릿 3개

예시:

- 잉여자금이 안정적으로 확보되어 있습니다.
- 비상금이 아직 목표 수준에 도달하지 않았습니다.
- 균형형 성향을 반영해 안정 저축과 적립식 운영을 함께 제안합니다.

정책:

- 이유는 최대 3개
- 숫자 기반 근거 1개 이상 포함 권장

#### F. Caution Section

노출 조건:

- 주의 포인트 존재 시

예시:

- 고정비 비중이 다소 높습니다.
- 비상금이 부족해 투자형 비중 확대는 아직 이릅니다.

정책:

- 경고 문구는 2개 이내로 압축
- 해결 가능한 톤으로 작성

#### G. Action Section

구성:

- PrimaryButton: `추천 계획 저장`
- SecondaryButton: `다시 진단하기`
- TextLink: `입력값 수정`

정책:

- 주 CTA는 1개만 강조
- 다시 진단하기보다 저장을 우선 액션으로 배치

## 5.6 결과 화면 예외 상태 상세안

### 예외 상태 목적

- 사용자가 추천 실패로 느끼지 않고 우선순위 전환으로 이해하게 만든다.

### 화면 구조

```text
[WarningBanner]
현재는 신규 저축보다 지출 구조 조정이 우선입니다

[Hero Summary Card]
권장 저축액 0원
월 잉여자금 0원
비상금 상태 없음

[Priority Action Card]
우선 조치 3가지

[Optional Recommendation Card]
추천 방식: 지출 조정 우선형

[Action Section]
입력값 수정
다시 진단하기
```

### Priority Action Card 예시

- 고정비 항목 다시 점검하기
- 변동지출 감축 가능 항목 찾기
- 비상금 1차 목표부터 설정하기

### UI 메모

- 예외 상태에서는 투자, 적립, ISA 같은 키워드를 전면에 노출하지 않는다.
- 해결 중심 정보가 추천보다 먼저 와야 한다.

## 6. 상태별 컴포넌트 정책

| 컴포넌트 | 기본 상태 | 선택 상태 | 에러 상태 | 비활성 상태 |
|---|---|---|---|---|
| NumberInputField | 빈 입력 | 값 입력 중 | 오류 메시지 노출 | 해당 없음 |
| SelectCard | 테두리 기본 | 배경 강조 + 체크 | 해당 없음 | 클릭 불가 |
| PrimaryButton | 활성 | 누름 상태 | 해당 없음 | 흐리게 표시 |
| WarningBanner | 숨김 | 노출 | 해당 없음 | 해당 없음 |

## 7. 인터랙션 메모

- 입력 필드 포커스 시 키보드에 가려지지 않도록 자동 스크롤 필요
- 목표/성향 선택은 탭 한 번으로 완료되어야 함
- 결과 화면 도달 시 상단부터 자연스럽게 읽히는 흐름이 중요함

## 8. 디자인 전달 메모

- 본 문서는 구조 우선 문서이므로 시각 디자인은 단순한 카드 기반 시스템으로 시작 가능
- 결과 화면은 숫자 대비와 정보 위계를 명확히 주는 것이 핵심
- 입력 화면은 브랜드 표현보다 부담 없는 작성 경험이 중요

## 10. 결론

MVP 로우파이 UI 상세안의 핵심은 최소 입력과 최대 이해를 동시에 만드는 것이다.  
특히 결과 화면은 이 서비스의 가치를 가장 직접적으로 보여주는 영역이므로, 숫자, 추천 방식, 이유, 실행 액션이 첫 화면 내에서 자연스럽게 연결되도록 설계해야 한다.
