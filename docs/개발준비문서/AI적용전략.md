# 가계부 기반 저축 운영 추천 서비스 AI 적용 전략

## 1. 문서 개요

본 문서는 가계부 기반 저축 운영 추천 서비스에 AI를 어떤 범위와 방식으로 적용할지 정의하기 위한 문서이다.  
본 문서의 목적은 MVP 단계에서 규칙 기반 로직과 AI 기반 보조 기능의 역할을 분리하고, 추후 자산관리 코치 기능으로 확장할 수 있는 기준을 마련하는 데 있다.

## 2. AI 적용 기본 원칙

- 정량 계산과 핵심 추천 판단은 규칙 기반으로 수행한다.
- AI는 계산 결과를 해석하고 설명하는 보조 레이어로 사용한다.
- AI가 직접 금융 판단의 최종 진실 소스가 되지 않도록 한다.
- 사용자에게 노출되는 AI 결과는 입력값과 규칙 기반 결과를 바탕으로 생성한다.
- 금융 권유로 오해될 수 있는 표현은 제한한다.

## 3. 기능 분리 기준

### 3.1 DB에 저장할 대상

- 사용자 입력 원본 데이터
- 계산 결과 스냅샷
- 추천 결과 스냅샷
- 월별 비교를 위한 핵심 이력 데이터

### 3.2 서버 규칙 로직으로 처리할 대상

- 월 잉여자금 계산
- 비상금 목표액 계산
- 비상금 충족률 계산
- 저축 여력 단계 분류
- 추천 방식 유형 결정
- 예외 상황 판별
- 추천 금액 배분 계산

### 3.3 AI로 보조할 대상

- 추천 이유 문구 생성
- 주의 포인트 문구 개인화
- 월별 리포트 요약 생성
- 자산관리 방향 코칭 문구 생성
- 사용자 질문에 대한 대화형 설명

## 4. MVP에서 AI를 적용하지 않는 범위

MVP에서는 아래 기능을 AI 없이 규칙 기반으로만 구현한다.

- 저축 가능액 계산
- 추천 방식 분기
- 예외 상황 판별
- 결과 화면 기본 문구

이유는 다음과 같다.

- 핵심 추천이 안정적으로 재현되어야 한다.
- 프롬프트나 모델 변경에 따라 추천 결과가 흔들리면 안 된다.
- 초기 단계에서는 추천 신뢰성과 일관성이 더 중요하다.

## 5. MVP 이후 AI 1차 적용 후보

### 5.1 추천 이유 문구 생성

가장 먼저 붙이기 좋은 영역이다.

입력 예시:

- 저축 여력 단계
- 비상금 상태
- 추천 방식 유형
- 권장 저축액

출력 예시:

- 현재는 비상금 보완이 우선이지만, 잉여자금이 안정적이어서 적금과 ISA 적립을 함께 가져가는 구조가 적절합니다.

적용 이유:

- 정량 계산은 규칙 기반으로 유지할 수 있다.
- AI는 사용자에게 더 자연스럽고 설득력 있는 설명을 제공할 수 있다.

### 5.2 주의 포인트 개인화

같은 예외 상태라도 사용자 상황에 맞게 표현을 다르게 할 수 있다.

예시:

- 고정비 비중이 높은 사용자
- 소비 변동성이 큰 사용자
- 사회초년생 사용자

적용 이유:

- 동일한 규칙 결과를 더 부드럽고 이해하기 쉬운 문장으로 전환할 수 있다.

### 5.3 월별 리포트 요약

월별 누적 데이터가 쌓이면 AI가 변화 요약을 생성할 수 있다.

예시:

- 지난달보다 잉여자금이 12만 원 줄었습니다.
- 변동지출 증가가 주요 원인입니다.
- 다음 달에는 생활비 상한을 먼저 정하는 것이 좋습니다.

적용 이유:

- 규칙만으로는 딱딱한 통계에 그치기 쉽다.
- AI는 변화 해석과 행동 제안을 자연스럽게 연결할 수 있다.

## 6. 중장기 AI 확장 후보

### 6.1 자산관리 방향 코칭

사용자의 월별 현금흐름과 축적된 이력을 기반으로 장기 자산관리 방향을 제안한다.

예시:

- 현재는 투자 확대보다 비상금 3개월치 확보가 우선입니다.
- 고정비 비중이 높은 상태이므로 절세 계좌 확대보다 지출 구조 조정이 먼저 필요합니다.
- 여유자금이 안정적으로 남고 있어 중기 목표 자금과 장기 운용 자금을 분리하는 것이 적절합니다.

적용 원칙:

- AI가 단독 판단하지 않는다.
- 규칙 기반 재무 상태 분류 결과를 받은 뒤 코칭 문구를 생성한다.

### 6.2 대화형 재무 코치

사용자가 추천 결과에 대해 질문하고 추가 설명을 받을 수 있도록 한다.

예시 질문:

- 왜 적금 비중이 더 높나요?
- 다음 달 수입이 줄면 추천이 어떻게 바뀌나요?
- 지금은 왜 ISA 비중이 낮은가요?

적용 원칙:

- 답변은 현재 추천 결과와 입력값을 근거로 생성한다.
- 새로운 금융 판단을 만들어내기보다 기존 판단을 설명하는 방향으로 제한한다.

## 7. AI 적용 시 주의사항

- 개별 종목 추천으로 이어지는 표현 금지
- 수익률 보장 또는 투자 확정 표현 금지
- 레버리지, 고위험 자산은 기본 추천축으로 사용 금지
- AI 출력은 반드시 규칙 기반 결과와 충돌하지 않아야 함
- 필요 시 AI 출력 앞뒤로 일반적 가이드 문구를 붙여 오해를 줄여야 함

## 8. 추후 데이터 구조 확장 후보

AI 고도화 시 아래 필드 또는 테이블을 추가로 고려할 수 있다.

- `ai_summary`
- `coaching_note`
- `asset_strategy_stage`
- `monthly_ai_report`
- `user_question_history`

단, MVP에서는 필수 저장 대상이 아니다.

## 9. 적용 우선순위

1. 추천 이유 문구 생성
2. 주의 포인트 문구 개인화
3. 월별 리포트 요약
4. 자산관리 방향 코칭
5. 대화형 재무 코치

## 10. 결론

이 서비스에서 AI는 계산 엔진을 대체하는 역할이 아니라, 규칙 기반 결과를 더 이해하기 쉽게 전달하고 장기적으로는 개인 맞춤 자산관리 코치 경험을 만드는 역할을 맡는 것이 가장 적절하다.  
따라서 MVP에서는 규칙 기반 추천의 신뢰성을 우선 확보하고, 이후 설명과 코칭 영역부터 단계적으로 AI를 적용하는 전략이 바람직하다.
