# 가계부 기반 저축 운영 추천 서비스 구현 현황

## 1. 문서 개요

본 문서는 현재 시점의 실제 구현 범위, 검증 완료 범위, 남은 작업을 정리하기 위한 문서이다.  
기획/개발준비문서와 실제 코드 구현 상태가 어디까지 맞춰졌는지 한 번에 파악할 수 있도록 작성한다.

## 2. 현재 구현 완료 범위 요약

현재 프로젝트는 MVP 핵심 흐름 기준으로 아래 범위까지 구현되었다.

- 백엔드 인증 API 구현 완료
- 백엔드 재무 입력 및 추천 결과 API 구현 완료
- 추천 계산/분기/예외 보정 엔진 구현 완료
- PostgreSQL + Prisma 저장 구조 연결 완료
- 최소 프론트 데모 화면 구현 완료
- e2e 계약 테스트 통과 완료

즉, 현재는 회원가입/로그인 후 월별 재무 정보를 입력하고 추천 결과를 저장/조회하는 기본 MVP 흐름이 동작하는 상태이다.

## 3. 백엔드 구현 현황

### 3.1 인증

완료:

- `POST /api/v1/auth/signup`
- `POST /api/v1/auth/login`
- `POST /api/v1/auth/refresh`
- `POST /api/v1/auth/logout`
- `GET /api/v1/auth/me`
- JWT access token 발급
- refresh token 발급 및 DB hash 저장
- `JwtAuthGuard` 적용

관련 파일:

- `apps/api/src/auth/*`
- `apps/api/src/repositories/user.repository.ts`

### 3.2 재무 입력 / 추천 API

완료:

- `POST /api/v1/financial-inputs`
- `GET /api/v1/financial-inputs/:inputId`
- `PUT /api/v1/financial-inputs/:inputId`
- `GET /api/v1/recommendations/:inputId`

특이사항:

- `POST`는 `userId + targetMonth` 기준 upsert
- `PUT`는 실제 `inputId` 기준 수정 후 추천 결과 재생성
- 모든 보호 API는 로그인 사용자 기준으로 동작

관련 파일:

- `apps/api/src/financial-inputs/*`
- `apps/api/src/recommendations/*`
- `apps/api/src/repositories/monthly-finance-input.repository.ts`
- `apps/api/src/repositories/recommendation-result.repository.ts`

### 3.3 추천 엔진

완료:

- `FinanceCalculator`
- `RecommendationEngine`
- `ExceptionPolicyEngine`
- `RecommendationActionFactory`

현재 동작 방식:

1. 입력값 저장
2. 잉여자금/고정비/비상금 계산
3. 추천 타입 분기
4. 예외 상황 보정
5. 추천 사유/주의 포인트/액션 조합
6. 추천 결과 저장
7. 결과 응답 반환

관련 파일:

- `apps/api/src/domain/calculators/finance-calculator.ts`
- `apps/api/src/domain/recommenders/recommendation-engine.ts`
- `apps/api/src/domain/policies/exception-policy-engine.ts`
- `apps/api/src/domain/factories/recommendation-action.factory.ts`

## 4. 데이터 계층 구현 현황

완료:

- PostgreSQL Docker 컨테이너 로컬 실행 확인
- Prisma schema 반영
- 초기 migration 적용
- Prisma 7 PostgreSQL adapter 적용

현재 사용 테이블:

- `users`
- `monthly_finance_inputs`
- `recommendation_results`

관련 파일:

- `apps/api/prisma/schema.prisma`
- `apps/api/prisma/migrations/*`
- `apps/api/src/database/prisma.service.ts`

## 5. 프론트 구현 현황

완료:

- Next.js 기반 단일 데모 페이지 구현
- 로그인 / 회원가입 폼
- 월별 재무 입력 폼
- 추천 결과 조회/표시
- 로컬스토리지 기반 토큰 저장
- 저장된 `inputId` 기준 결과 재조회 버튼

현재 상태:

- 한 페이지에서 인증 -> 입력 -> 결과 확인까지 가능
- 데모 성격의 단일 페이지 구조
- 문구는 현재 영어 중심 UI 상태

관련 파일:

- `apps/web/src/app/page.tsx`
- `apps/web/src/app/layout.tsx`
- `apps/web/src/app/globals.css`
